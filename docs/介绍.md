# DokePHP 介绍

DokePHP 是一个纯 PHP 的迷你 API 框架，目标是兼容 PHP 5.6 - 8.3，并且不依赖 Composer 与任何第三方库。它强调“内核/业务分离”、可观测性与安全默认值，适合快速交付中小型接口服务与内部后台 API。

## 设计目标

- 零依赖：不使用 Composer，不引入第三方库
- 兼容性：PHP 5.6 - 8.3
- 可上线：统一错误码、请求链路追踪、结构化日志、限流、请求体大小限制、安全响应头
- 可扩展：中间件管线、路由分组、依赖注入容器、服务层组织方式

## 核心能力

- 路由：支持路径参数 `/users/{id}`，支持路由分组与路由级中间件
- 中间件：全局中间件 + 路由级中间件，支持 `alias:param` 参数化
- 请求/响应：统一 JSON 返回，支持 HEAD/OPTIONS/CORS 预检
- 异常处理：统一捕获并输出稳定的 JSON 错误结构（debug 可输出详细 trace）
- 日志：
  - access 日志：`storage/logs/access.jsonl`（含 request_id、耗时、状态码）
  - app 日志：`storage/logs/app.jsonl`（异常与应用日志）
  - audit 日志（可选）：`audit_logs` 表或 `storage/logs/audit.jsonl`
- 安全：
  - 安全响应头（CSP、Referrer-Policy、Permissions-Policy 等）
  - Webhook 签名验签（HMAC + timestamp/nonce + 防重放）
  - 请求体大小限制（413）、限流（429）
- 鉴权与权限：
  - Bearer Token：file/db/hybrid 三种驱动
  - RBAC：config/db/hybrid 三种驱动，支持 `permission:*`、`prefix*` 与精确匹配
- 数据库：
  - PDO 封装与事务
  - 迁移系统（migrate/status/rollback）
  - 分页（Db::paginate）

## 适用场景

- 内部管理后台/运维后台的 API（账号、角色权限、审计）
- 中小型业务的接口服务（CRUD + 校验 + 分页 + 鉴权 + RBAC）
- 网关后的单一职责服务（用户中心、权限中心、通知服务等）
- 需要“可控、不依赖外部生态”的交付环境（受限内网、镜像受限、无 Composer 环境）

# DokePHP 使用手册（从 0 开始）

## 1. 启动与环境

将 `Public/` 配置为站点根目录。

本地开发推荐使用 PHP 内置 Server，并且必须指定 router script：

```bash
APP_ENV=dev php -S 127.0.0.1:8000 -t Public Public/router.php
```

如果只写 `-t Public`，类似 `/admin/stats` 这类“非真实文件路径”不会进入入口文件，可能直接 404。

## 2. 配置管理（建议使用 app.local.php 覆盖）

配置目录：`App/Config/`

加载顺序（从低到高覆盖）：

1. `App/Config/*.php`（基础配置）
2. 若设置 `APP_ENV=dev`：加载 `app.dev.php` 覆盖
3. 若存在 `app.local.php`：最后覆盖（本地/线上私有配置，不入库）

生成 `app.local.php`（推荐）：

```bash
php bin/make-local.php --force \
  --app.debug=true \
  --db.dsn="mysql:host=127.0.0.1;dbname=test;charset=utf8mb4" \
  --db.username=root --db.password=pass \
  --auth.enabled=true --auth.driver=db \
  --rbac.enabled=true --rbac.driver=db \
  --audit.enabled=true
```

## 3. 数据库迁移

迁移目录：`database/migrations/`

常用命令：

```bash
php bin/migrate.php migrate
php bin/migrate.php status
php bin/migrate.php rollback 1
```

## 4. 路由与控制器开发

路由文件：`App/Routes/web.php`

- 支持：`get/post/add('PUT', ...)`、`group(prefix/middleware, callback)`
- handler 支持 `Class@method` 或 callable

控制器建议保持“薄”，把业务逻辑放到 `App/Services/*Service.php`。

参数校验建议统一用：

- `Framework/Foundation/BaseController::validate($data, $rules, $messages)`
- 校验失败会抛 `ValidationException` 并以 422 返回字段错误

## 5. 统一 JSON 返回与错误码

- 成功：`Api::ok(data, message)`
- 失败：`Api::fail(message, business_code, http_status, data)`

错误码定义：`Framework/Support/ErrorCodes.php`

## 6. 鉴权（Bearer Token）

开启鉴权：

- 全局鉴权：设置 `auth.enabled=true`，并配置 `auth.driver`
- 路由级强制鉴权：对路由组加 `auth_required`

三种驱动：

- `file`：从配置 `auth.tokens` 或 `auth.token_file` 读取
- `db`：从 `auth_tokens` 表读取（只存 `sha256(token)`，不存明文）
- `hybrid`：兼容旧 file 与新 db（推荐灰度）

## 7. RBAC（权限控制）

开启 RBAC：

- `rbac.enabled=true`
- `rbac.driver=config|db|hybrid`

路由上挂：

- `permission:admin`
- `permission:order.read`

后台管理接口位于 `/admin/*`，默认要求：

- `auth_required`
- `rbac_required`（RBAC 未启用直接 403）
- `permission:admin`

建议先初始化 admin：

```bash
php bin/migrate.php migrate
php bin/seed-rbac.php admin 0
```

## 8. 审计（可选）

启用：

- `audit.enabled=true`
- 执行审计迁移（audit_logs）

后台写操作与 401/403 会记录审计，可通过：

- `GET /admin/audit`
- `GET /admin/audit?action=admin.tokens.rotate`

## 9. RBAC 导入导出（备份/迁移）

导出：

```bash
php bin/export-rbac.php storage/cache/rbac-export.json
```

导入（先 dry_run 看影响）：

```bash
php bin/import-rbac.php storage/cache/rbac-export.json --mode=merge --purge=0 --dry_run=1
php bin/import-rbac.php storage/cache/rbac-export.json --mode=merge --purge=0
```

## 10. 上线 checklist（强建议）

- `app.debug=false`
- 按业务收紧 CORS（避免长期 `allow_origin=*`）
- WebServer 层禁止访问 `storage/`（避免日志外泄）
- 配置 `body_limit.max_bytes` 与 `rate_limit`（防御滥用）
- 生产使用 Nginx/Apache 指向 `Public/`，不要用内置 server
